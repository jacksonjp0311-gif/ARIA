# ARIA Roadmap

## 0.1 — Bootstrap proof

- deterministic parser, semantic validator, bytecode compiler, container, VM;
- explicit memory and glyph graph metadata;
- deny-by-default filesystem capabilities;
- PowerShell 5.1 execution and cross-platform PowerShell CI.

## 0.2 — Typed language core

- type declarations and algebraic data types;
- functions, branches, iteration, structured errors;
- lexical scopes and immutable bindings;
- formal bytecode verifier;
- conformance fixtures independent of the reference compiler.

## 0.3 — Graph execution

- graph pattern matching and guarded rewrite rules;
- event streams and provenance history;
- rollback transactions;
- capability tokens with explicit lifetime and delegation.

## 0.4 — Glyph studio

- lossless text/glyph round-trip;
- accessible glyph palette;
- live execution graph and time-travel inspection;
- source control diff expressed as semantic graph changes.

## 0.5 — AI bridge

- provider-neutral proposal protocol;
- context packaging from graph and memory;
- simulation and approval workflows;
- test and evidence requirements.

## 1.0 — Open language contract

- stable specification and standard library;
- independent VM implementation;
- package format and registry rules;
- portable target such as WebAssembly;
- security review and compatibility suite.

Self-hosting is a possible later milestone, not a prerequisite for language legitimacy.
